# Melodia – Cloud Music & Smart Playlist Companion

Melodia is a premium, Spotify-inspired music streaming platform built using the MERN stack (React 19 + Node/Express + MongoDB) with a responsive glassmorphic UI, Howler.js for seamless audio streaming, Cloudinary storage integrations, and an AI-driven smart playlist recommendation assistant.

---

## Features

- **Spotify-Inspired Theme:** Glassmorphic cards, black backdrop, purple/pink neon glows, and custom scrollbars.
- **Audio Deck Control:** fixed bottom deck with play/pause, skip, volume sliding, seek sliding, mute, shuffle, and repeat.
- **AI Smart Playlist Generator:** Analyzes liked songs, favorite genres, and play histories to build personalized thematic mixes ("Chill Vibes", "Workout Mix", "Night Drive", "Top Picks For You").
- **Live Filter Search:** Instantly query by keywords or filter by explicit artist names, genres, moods, or track lengths.
- **Library Dashboard:** Centralized panel for liked tracks, custom-created playlists, smart mixes, and listening history logs.
- **Admin Management Portal:** Secured route allowing catalog additions, direct uploads of cover images and MP3 files to Cloudinary, track updates, and deletion controls.

---

## Tech Stack

### Frontend
- **React 19** + **Vite** (Rapid builds and fast dev-server)
- **Tailwind CSS v3** + **PostCSS** (Custom glassmorphic utility layer)
- **React Router DOM** (Unified client-side routing)
- **Axios** (Configured HTTP clients with token interceptors)
- **Framer Motion** (Micro-interaction UI animations)
- **Howler.js** (HTML5 audio streaming engine)
- **Context API** (State management for Auth, Player, and Playlists)

### Backend
- **Node.js** + **Express.js** (REST API)
- **MongoDB Atlas** + **Mongoose** (Database schemas)
- **JWT Authentication** + **Bcrypt.js** (Security and session caching)
- **Multer** + **Cloudinary** (Audio and cover art storage upload)

## Environment Variables Setup

### Backend Environment Configuration
Create a `.env` file in the `backend/` directory:

```env
PORT=5000
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.xxxxxx.mongodb.net/melodia?retryWrites=true&w=majority
JWT_SECRET=your_jwt_secret_key_here_melodia_companion_app
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
NODE_ENV=development
```

### Frontend Environment Configuration
Create a `.env` file in the `frontend/` directory:

```env
VITE_API_URL=http://localhost:5000/api
```

## Local Setup & Installation

### Prerequisite
Ensure you have **Node.js (v18+)** and **npm** installed on your system.

### Step 1: Install Dependencies
Run the install command in both directories:

**Backend:**
```bash
cd backend
npm install --legacy-peer-deps
```

**Frontend:**
```bash
cd frontend
npm install
```

### Step 2: Seed the Database
Configure a valid `MONGODB_URI` inside `backend/.env` first, then run the database seed script to populate 20 tracks, 10 artists, 5 genres, and default users:
```bash
cd backend
npm run seed
```

*Note: Default users generated by seed:*
- **Admin User:** Email: `admin@melodia.com` | Password: `adminpassword`
- **Tester User:** Email: `tester@melodia.com` | Password: `testpassword`

### Step 3: Run the Servers
Open two terminal windows to run both servers locally.

**Backend dev server:**
```bash
cd backend
npm run dev
```
*(Runs on [http://localhost:5000](http://localhost:5000))*

**Frontend dev server:**
```bash
cd frontend
npm run dev
```
*(Runs on [http://localhost:3000](http://localhost:3000))*

## Deployment Steps

### Frontend Deployment (Vercel)
1. Initialize Git in the frontend directory or root and push to GitHub.
2. Link your repository in Vercel.
3. In the Vercel project configuration, set the **Framework Preset** to `Vite`.
4. Configure the **Build Command** to `npm run build` and the **Output Directory** to `dist`.
5. Add the Environment Variable:
   - `VITE_API_URL`: Your deployed Render API root (e.g. `https://melodia-api.onrender.com/api`).
6. Deploy!

### Backend Deployment (Render)
1. Create a new **Web Service** on Render and link your GitHub repository.
2. Configure build settings:
   - **Runtime:** `Node`
   - **Build Command:** `npm install --legacy-peer-deps`
   - **Start Command:** `node server.js`
3. Add the following Environment Variables in the "Environment" tab:
   - `MONGODB_URI`: Your MongoDB Atlas connection string.
   - `JWT_SECRET`: Secure cryptographic token string.
   - `CLOUDINARY_CLOUD_NAME`: Cloudinary Cloud Name.
   - `CLOUDINARY_API_KEY`: Cloudinary API Key.
   - `CLOUDINARY_API_SECRET`: Cloudinary API Secret.
   - `NODE_ENV`: `production`
4. Click **Deploy Web Service**.
